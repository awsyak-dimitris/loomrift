# CLAUDE.md — Loomrift Project Context

## What is Loomrift?
A visual operating system for tracking people, teams and projects as an interactive 3D/2D graph.
Single-file HTML app deployed to GitHub Pages via GitHub Contents API.

## Architecture
- **Single HTML file** (`index.html`) with embedded CSS + JS
- **Three.js r128** (CDN) for 3D graph rendering
- **Two canvas layers**: `lc` (2D overlay) z-index:1, `c` (Three.js) z-index:2
- **localStorage** persistence with versioned key (`loomrift_v9`)
- **No build system** — file is generated by `build2.py`

## Known Pitfalls
- `var` hoisting causes blank screens when same variable declared twice in mkTex
- `confirm()` blocked by GitHub Pages iframe policy
- `source-in` composite must happen BEFORE fill overlay, not after
- Three.js CanvasTexture objects cannot be cached across materials
- Python `\` line continuations don't work in JS string literals
